2026.4.20

本次更新对游戏底层架构进行了全面彻底的重构，解决了以往代码臃肿、耦合度高的问题，极大地提升了系统的可维护性、运行性能以及联机同步的稳定性。

🏗️ 核心架构大修 (Architecture Overhaul)

移除 Game.ts God Class (剥离上帝类)：
大幅减轻了原 Game.ts 的负担，将其庞杂的职能拆分并下放至多个职责单一的实体管理器（如 CombatManager, MobManager, ResourceManager, CardpileManager, CursorManager 等）。各子系统间现在通过低耦合的事件信号进行通信调度。

React UI 与 Phaser 引擎完全解耦：
彻底移除了 .tsx（React 组件）中对 Phaser 游戏实例的直接引用。现在所有的 UI 组件仅通过发送全局事件信号（Signal）来驱动游戏内部逻辑，并监听来自游戏引擎的读信号进行单向状态更新，实现了完美的“单向数据流”。

事件总线系统类型安全化 (EventBus Refactor)：
废弃了原先脆弱的硬编码字符串事件名。现已全面引入类型安全的 EVENT_* 常量（如 EVENT_RoomGameStart, EVENT_EnergyUpdate 等共 25 个），提供完整的参数定义和 JSDoc 注释，杜绝了拼写错误并大幅提升了 IDE 的代码提示体验。

👾 核心实体系统重构 (Entity System)

引入 Model - Entity 状态/渲染分离架构：
全面重构了植物、僵尸、发射物等游戏内存实体。

逻辑层 (Model)：如 PlantModel，全权负责生命值、伤害、资源生成等纯游戏逻辑，以及利用 TickManager 掌控基于严格游戏帧的事件回调（onCreate, onStarShards），确保联机状态下的一致性。

渲染层 (Entity)：如 PlantEntity，专注于纯粹的表现工作——视图构建（buildView）、基于真实时间的动画过渡控制（delayedCall）等，使得游戏的渲染表现不再与底层逻辑深度绑定。

实体初始化规范化：
为实体对象实现了标准的 initializeEntity 生命周期方法，统一了视图坐标和物理碰撞体的建立流程。

优化实体坐标系中心：
优化了植物和障碍物的坐标换算，现统一使用物理碰撞体中心（Body Center）进行精准定位对齐。

✨ 视觉表现与游戏体验 (Visual & Gameplay)

UI 操作反馈增强：
新增资源匮乏时的直观视觉反馈。当玩家在“星之碎片”不足时尝试使用相关消耗能力，操作栏会触发显眼的红色闪烁提示。

卡牌互动逻辑重置：
重构了底部卡槽、选卡功能的底层交互逻辑，精简了内部组件的 Props 接口，植物种子的选取与部署动作变得更加灵敏。

内置菜单与暂停视图迭代：
增强了局内暂停菜单 (PauseMenu) 及底层内部菜单的显示排版，游戏进程的挂起/恢复判定更加安全。

🐛 修复与优化 (Fixes & Optimizations)

联机状态流转强化：深度优化了联机控制 OnlineStateManager 及房间通讯。实装了更严谨的 RoomAllReadyEvent 判断，确保多端玩家彻底就绪后再进入同地图帧同步。

Mock Server 本地服务器升级：对测试框架的模拟服务器进行了功能补全，即使在离线单机模式下，也能充分运行并测试全套重构后的 WebSocket 联机时序逻辑。
